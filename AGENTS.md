# 仓库协作说明

本文件面向在本仓库内维护提示词的 AI Agent 与协作者。

## 仓库目标

本仓库用于维护《卡拉彼丘》（Strinova）角色扮演提示词，以及可跨角色复用的世界观补充文档。

维护目标有三点：

1. 角色设定尽量贴近官方资料
2. 提示词结构稳定，便于直接投喂模型
3. 共通背景集中维护，减少重复和设定漂移

## 目录约定

- `personas/*.md`：单个角色提示词
- `personas/TEMPLATE.md`：新增角色时的统一模板
- `world/*.md`：跨角色复用的世界观补充
- `examples/*.md`：可直接复用的提示词组合示例
- `scripts/validate.sh`：最小结构校验脚本
- `SOURCES.md`：来源登记总表与维护状态说明
- `README.md`：对外说明、使用方式、角色索引
- `AGENTS.md`：仓库协作说明，内容必须使用中文
- `CLAUDE.md`：Claude Code 专用入口，内容固定为 `@AGENTS.md`

## 信息来源优先级

更新角色或世界观时，按下面顺序取材：

1. 角色主词条
2. 语音台词、誓约、羁绊剧情、初识剧情
3. 技能、武器、时装、玩法页
4. `world/` 目录中的共通补充

处理规则：

- 主词条优先于补充页
- 角色个人资料优先于共通世界观
- 若不同页面表述冲突，优先采用更新且更直接的页面
- 若官方资料未写明，保持保守，不要补成硬设定

统一参考入口：

- `https://wiki.biligame.com/klbq/`

## 角色文件结构

角色文件默认沿用现有结构，常见段落如下：

- `任务`
- `设定基准`
- `角色档案`
- `角色核心`
- `与用户的默认关系`
- `语言风格`
- `角色口吻要点`
- `战斗与游戏设定`
- `官方口吻参考` 或 `语感参考`
- `适合[角色名]提及的细节`
- `示例对话风格`
- `时装参考`
- `禁止事项`
- `最终执行准则`

除非仓库整体模板调整，否则不要随意打乱主要标题顺序。
新增角色默认从 `personas/TEMPLATE.md` 起稿。

## 写作规范

- 使用中文
- 语气自然、口语化，不写成生硬翻译腔
- 单句尽量控制在 35 字内
- 不使用括号描写动作、心理或系统提示
- 优先保留角色稳定特征，不靠夸张语气制造“像”
- 默认关系可以亲近，但不要直接写成失控恋爱、病娇或极端同人化
- 不编造开放时间、活动轮换、版本改动等易过期信息

## 世界观文档规范

`world/` 目录只处理跨角色复用的信息，不替代角色个人设定。

编辑时注意：

- 只写多个角色都会用到的背景
- 角色个体经历优先放回对应角色文件
- 若某条信息只能帮助理解气氛，但缺少强依据，应明确标注使用边界
- 不要让 `world/` 中的补充压过角色主词条

## 修改流程

编辑前：

1. 先阅读现有同类文件，至少找 2 到 3 份相近角色作参考
2. 确认仓库中是否已有可复用表述或世界观补充
3. 明确这次修改是补资料、修口吻、补结构，还是同步新角色

编辑时：

1. 小步修改，尽量保持标题和段落结构稳定
2. 能复用现有表述就复用，避免重复发明格式
3. 新增角色时，先复制 `personas/TEMPLATE.md`，再替换角色名、来源链接和口吻细节
4. 涉及共通背景时，优先补 `world/`，再回填角色文件引用范围

编辑后：

1. 检查角色名、阵营名、链接和称呼是否一致
2. 检查是否引入了无依据扩写
3. 若新增或重分类角色，记得同步 `README.md` 与 `SOURCES.md`
4. 若新增 `world/` 文档，记得同步 `world/README.md`
5. 运行 `bash scripts/validate.sh`
6. 确认 `CLAUDE.md` 仍然只包含 `@AGENTS.md`

## 提交前检查

- 文档为中文
- 设定与官方资料一致
- 无明显同人扩写或过期信息
- 章节结构清晰
- 链接可用，文件名正确
- `SOURCES.md` 已同步
- `bash scripts/validate.sh` 已通过
- `README.md`、`AGENTS.md`、`CLAUDE.md` 三者关系保持一致
