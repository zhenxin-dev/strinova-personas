# 组合示例

这份文档给出几组可以直接套用的提示词组合。

总原则：

- 先放 `1` 个角色文件
- 再补 `0 到 2` 个 `world/` 文件
- 世界观文件只负责托底，不要抢角色本人

## 示例一：日常聊天

适合场景：

- 下午茶
- 轻委托
- 稍微亲近一点的陪伴聊天

System:

- `personas/奥黛丽.md`
- `world/世界观概览.md`

User:

```text
会开完了吗？我带了新买的司康，想顺便和你聊聊下个委托。
```

使用重点：

- 先落在奥黛丽和用户的私人互动
- 合作、计划和品位作为自然延伸
- 不要一上来就把乌尔比诺背景讲成长段设定

## 示例二：战斗复盘

适合场景：

- 爆破复盘
- 站位提醒
- 保护欲偏强的任务交流

System:

- `personas/信.md`
- `world/术语速查.md`

User:

```text
刚才那局 B 点我站位太靠前了。你直接说，我下把该怎么改？
```

使用重点：

- 让信先给结论，再补理由
- `弦化`、站位、推进等术语只点到为止
- 重点是职责感和保护感，不是战术百科

## 示例三：疗愈与场景感

适合场景：

- 训练后安抚
- 轻医疗感对话
- 日常陪伴与状态确认

System:

- `personas/星绘.md`
- `world/地点索引.md`
- `world/写作边界.md`

User:

```text
今天训练强度有点高，我脑子还在嗡嗡响。你陪我缓一会儿吧。
```

使用重点：

- 优先写星绘的照顾感和说话节奏
- `星庇所` 这类地点只拿来稳住氛围
- 不要把模糊的记忆设定扩成完整科幻说明

## 示例四：旧关系与立场冲突

适合场景：

- 兄弟旧事
- 阵营分歧
- 较严肃的角色对话

System:

- `personas/令.md`
- `world/阵营关系.md`
- `world/人物关系索引.md`

User:

```text
如果有一天你和信必须站到同一现场，你会先看立场，还是先看他有没有受伤？
```

使用重点：

- 先写令本人怎么看这件事
- 再带出兄弟关系和阵营裂痕
- 重点是沉默、判断和保护，不是狗血撕裂戏

## 示例五：舞台与私下反差

适合场景：

- 通告后闲聊
- 偶像营业感逐渐放下
- 轻微心动和试探

System:

- `personas/香奈美.md`
- `world/世界观概览.md`

User:

```text
今天台上很闪。现在下了舞台，你想先当偶像，还是先当你自己？
```

使用重点：

- 先保住香奈美的甜和亮
- 再慢慢露出私下那层认真与小恶魔感
- 不要写成全程营业，也不要写成廉价撩拨

## 示例六：角色切换

如果要写多角色串场，不建议把两份完整角色词硬塞进同一个 system。

更稳的做法是：

1. 先用一个角色文件完成当前场景
2. 切换角色时更换 system
3. 只保留上一轮剧情摘要，不同时挂两份完整人格词

场景 A System:

- `personas/白墨.md`
- `world/地点索引.md`

场景 A User:

```text
带我在 88 区转转。别走观光路线，我想看你平时真会去的地方。
```

场景 B System:

- `personas/汐.md`
- `world/人物关系索引.md`

场景 B User:

```text
白墨刚把委托内容转给你了。你先别急着答应，跟我说说这单值不值。
```

使用重点：

- 白墨和汐分别保留自己的说话底色
- `长期委托` 只用来解释合作关系，不自动升级成绑定关系
- 场景切换时靠摘要承接，不靠把角色词混在一起

## 维护场景示例

新增角色时，建议这样走：

1. 复制 `personas/TEMPLATE.md`
2. 先补 `## 设定基准`
3. 再写角色核心、默认关系和口吻部分
4. 更新 `README.md` 和 `SOURCES.md`
5. 运行 `bash scripts/validate.sh`
